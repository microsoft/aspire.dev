Aspire provides a first-class setup experience for AI coding agents. Run `aspire agent init` in your project and your AI assistant — whether it's GitHub Copilot, Claude Code, or another <abbr title="Model Context Protocol" data-tooltip-placement="top">MCP</abbr>-compatible tool — can immediately understand, build, debug, and monitor your distributed applications.

## Why Aspire for coding agents

Aspire gives coding agents the same visibility into your running application that a developer has. The resource data, structured logs, and distributed traces you see in the [Aspire Dashboard](/dashboard/overview/) are also reachable from the [Aspire CLI](/get-started/install-cli/) — so whether a person is debugging in the dashboard or an agent is diagnosing from a prompt, they see the same picture.

The Aspire CLI is built for agent-driven workflows — every command works non-interactively without blocking on prompts and supports `--format Json` for structured plain text output. Key commands include `aspire start` (background execution), `aspire start --isolated` (parallel worktrees), `aspire wait` (block until healthy), `aspire describe`, `aspire logs`, and `aspire docs search`.

The **Aspire skill file** (installed by `aspire agent init`) is the primary way agents learn these patterns. The skill file tells your agent when and how to use each CLI command, so it can build, start, debug, and monitor your distributed app without any additional runtime wiring.

## Get started

When you create a new Aspire project with `aspire new` or `aspire init`, you're prompted to configure AI agent environments. You can also run `aspire agent init` at any time to set up or update the configuration.

<Steps>

1. Open a terminal in your Aspire project directory (the folder containing your AppHost).

1. Run the following command:

   ```bash title="Aspire CLI"
   aspire agent init
   ```

1. Select the components you want to configure:

   - **Aspire skill file** (recommended) — teaches your AI agent how to use Aspire CLI commands
   - **Playwright CLI** — enables browser automation for testing web resources

   The Aspire [MCP server](/get-started/aspire-mcp-server/) is also offered as a legacy component. Most projects should use the skill file instead — see the [note below](#skills-vs-mcp-server).

</Steps>

:::caution[Skills vs. MCP server]
The Aspire skill file and the Aspire MCP server cover overlapping ground, and enabling both at the same time leads to duplicate tools, conflicting guidance, and confused agents. **Pick one — and for new projects, pick the skill file.** If you previously configured the Aspire MCP server, re-run `aspire agent init`, enable the skill file, and remove the `aspire` entry from your MCP configuration. See [Migrate from the Aspire MCP server](#migrate-from-the-aspire-mcp-server).
:::

## What gets configured

The `aspire agent init` command detects your AI development environment and creates the appropriate configuration files:

### Aspire skill file

The skill file teaches your AI coding agent how to work with Aspire. It includes a CLI command reference, key workflows (starting apps, debugging issues, adding integrations), and important rules to follow. The file is created in the format your agent expects:

<FileTree>

- .github/skills/aspire/
  - SKILL.md  GitHub Copilot
- .claude/skills/aspire/
  - SKILL.md  Claude Code

</FileTree>

The skill file guides your agent on how to:

- Start and manage your Aspire application (`aspire start`, `aspire stop`, `aspire describe`)
- Debug issues using structured logs and distributed traces
- Add integrations with `aspire add` and search documentation with `aspire docs`
- Use resource MCP tools for database queries and other resource-specific operations

### Aspire MCP server (legacy)

The [Aspire MCP server](/get-started/aspire-mcp-server/) is a legacy integration path that exposes resource status, logs, traces, and commands to your agent over MCP. It remains available for teams that have specific reasons to use MCP directly, but it is **not recommended alongside the skill file** — the two mechanisms overlap and will confuse the agent if both are active.

For new projects, skip the MCP server and use the skill file. If you need MCP for a specific workflow, see the [Aspire MCP server](/get-started/aspire-mcp-server/) reference.

## Migrate from the Aspire MCP server

If your project was previously configured with the Aspire MCP server (via an earlier `aspire agent init` run, or by hand), switch to the skill file:

<Steps>

1. Re-run `aspire agent init` and select **Install Aspire skill file** (deselect the Aspire MCP server).

2. Open your agent's MCP configuration and remove the `aspire` entry:

   - VS Code: `.vscode/mcp.json`
   - Claude Code: `.mcp.json`
   - Copilot CLI: `~/.copilot/mcp-config.json`
   - OpenCode: `opencode.jsonc`

3. Restart your agent so it picks up the new skill file and drops the MCP tools.

</Steps>

The skill file covers the same workflows the MCP server did (inspecting resources, tailing logs, querying traces, running commands) by driving the Aspire CLI — which keeps the agent working the same way a developer does.

## Migrate from AGENTS.md

If your project has an `AGENTS.md` file from a previous version of Aspire, you can migrate to the new skill file format. The skill file provides better structure and is recognized natively by AI coding agents.

<Steps>

1. Run `aspire agent init` and select **Install Aspire skill file**.

2. The skill files are created at `.github/skills/aspire/SKILL.md` and `.claude/skills/aspire/SKILL.md`.

3. Review and delete the old `AGENTS.md` file — the skill file replaces it.

</Steps>

:::tip
The skill file is more structured than `AGENTS.md` and includes a complete CLI command reference table, workflow patterns, and rules that the agent follows automatically. You can customize it for your project's specific needs.
:::

## Supported AI assistants

The `aspire agent init` command supports the following AI assistants:

- [VS Code](https://code.visualstudio.com/docs/copilot/customization/mcp-servers) with GitHub Copilot
- [Copilot CLI](https://docs.github.com/en/copilot/how-tos/use-copilot-agents/use-copilot-cli#add-an-mcp-server)
- [Claude Code](https://docs.claude.com/en/docs/claude-code/mcp)
- [OpenCode](https://opencode.ai/docs/mcp-servers/)

:::note
The skill file format (`SKILL.md`) is recognized by a growing number of agents. `aspire agent init` writes files in the locations each supported agent expects, and additional environments can be added over time.
:::
