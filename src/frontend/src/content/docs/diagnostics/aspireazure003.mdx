> Azure experimental types and members are for evaluation purposes only and are subject to change or removal in future updates. Suppress this diagnostic to proceed.

The `ASPIREAZURE003` diagnostic is emitted when your AppHost uses an experimental Azure hosting API. The following APIs currently carry this diagnostic:

- **Azure Virtual Networks** (`Aspire.Hosting.Azure.Network`): APIs such as `AddAzureVirtualNetwork` for modeling private networking in your app model.
- **`ReferenceRoleAssignmentAnnotation`** (`Aspire.Hosting.Azure`): An annotation that expresses implied role assignments when one resource references another across compute environments. Used internally when a resource references a Foundry hosted agent from a different compute environment to auto-wire the **Azure AI User** RBAC grant.

## Examples

### Azure Virtual Networks

The following code generates `ASPIREAZURE003`:

```csharp title="C# — AppHost.cs"
var vnet = builder.AddAzureVirtualNetwork("vnet");
var subnet = vnet.AddSubnet("pe-subnet", "10.0.1.0/24");
```

### Cross-compute-environment hosted agent reference

Aspire uses `ReferenceRoleAssignmentAnnotation` internally when a resource references a Foundry hosted agent from a different compute environment. That annotation lets Azure publishing auto-wire the **Azure AI User** RBAC grant on the owning Foundry account. Public AppHost APIs such as `AsHostedAgent`, `WithReference`, and `WithComputeEnvironment` don't require you to suppress `ASPIREAZURE003`.

For more information, see [Reference a hosted agent across compute environments](/integrations/cloud/azure/azure-ai-foundry/azure-ai-foundry-host/#reference-a-hosted-agent-across-compute-environments).

## To correct this error

Suppress the error with either of the following methods:

- Set the severity of the rule in the _.editorconfig_ file.

  ```ini title=".editorconfig"
  [*.{cs,vb}]
  dotnet_diagnostic.ASPIREAZURE003.severity = none
  ```

  For more information about editor config files, see [Configuration files for code analysis rules](/diagnostics/overview/#suppress-in-the-editorconfig-file).

- Add the following `PropertyGroup` to your project file:

  ```xml title="C# project file"
  <PropertyGroup>
      <NoWarn>$(NoWarn);ASPIREAZURE003</NoWarn>
  </PropertyGroup>
  ```

- Suppress in code with the `#pragma warning disable ASPIREAZURE003` directive.
