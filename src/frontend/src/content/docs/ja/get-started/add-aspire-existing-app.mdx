既存のアプリケーションに Aspire を追加する際、最初に選ぶのは AppHost の言語です。C# または TypeScript の AppHost を選択することで、使い慣れたツールチェーンのまま、既存のアプリケーションに対してサービスのオーケストレーション、リソースの定義、関係性のモデリングを行うことができます。

<Aside type="note" title="アプリの言語と AppHost の言語は、それぞれ別に選択できます">
オーケストレーション層で使用したいツールチェーンに合わせて AppHost の言語を選択してください。どちらの AppHost でも、複数の言語で作られたアプリをオーケストレーションできます。
</Aside>

## 始める前に

いずれの手順に進む前に、 [事前準備](/ja/get-started/prerequisites/) を完了させ、 [Aspire CLI をインストール](/ja/get-started/install-cli/)していることを確認してください。

コマンドの詳細や言語オプションについては、 [CLI リファレンス: `aspire init`](/reference/cli/commands/aspire-init/) をご覧ください。
