This guide walks you through migrating an existing C# AppHost (either project-based or file-based) to a TypeScript AppHost. By the end, you'll have an `apphost.ts` that replicates the same resource setup as your existing `AppHost.cs`.

<Aside type="caution" title="Source control first">
Before starting, ensure your existing project is committed to source control. This migration involves renaming or removing files, and you'll want to be able to reference your original configuration.
</Aside>

## Prerequisites

This guide requires Aspire 13.2 or later with TypeScript AppHost support enabled. See the [TypeScript AppHost prerequisites](/app-host/typescript-apphost/#prerequisites) for the full list of required tools and versions.

## Inventory your existing AppHost

Before migrating, take inventory of your current setup. You'll need this information to recreate your resources in TypeScript.

<Steps>

1. **Note your package references**

   Depending on your AppHost type, find your hosting integration packages:

   **Project-based AppHost** — check your `.csproj` file for `PackageReference` entries:

   ```xml title="AppHost.csproj"
   <ItemGroup>
     <PackageReference Include="Aspire.Hosting.Redis" Version="13.1.2" />
     <PackageReference Include="Aspire.Hosting.PostgreSQL" Version="13.1.2" />
     <PackageReference Include="Aspire.Hosting.NodeJs" Version="13.1.2" />
   </ItemGroup>
   ```

   **File-based AppHost** — check your `AppHost.cs` for `#:package` directives:

   ```csharp title="AppHost.cs (file-based)"
   #:package Aspire.Hosting.Redis@13.1.2
   #:package Aspire.Hosting.PostgreSQL@13.1.2
   #:package Aspire.Hosting.NodeJs@13.1.2
   ```

   Note each full package name—you'll use these directly with `aspire add`.

2. **Document your resource definitions**

   Copy or screenshot your current resource definitions from `AppHost.cs`. You'll reference these when writing the TypeScript equivalent. For example:

   ```csharp title="C# — AppHost.cs"
   var builder = DistributedApplication.CreateBuilder(args);

   var cache = builder.AddRedis("cache");
   var db = builder.AddPostgres("pg")
       .AddDatabase("catalog");

   var api = builder.AddProject<Projects.Api>("api")
       .WithReference(cache)
       .WithReference(db);

   builder.AddProject<Projects.Web>("web")
       .WithReference(api)
       .WithExternalHttpEndpoints();

   builder.Build().Run();
   ```

3. **Save your `.aspire/settings.json`**

   If you have a local or workspace `.aspire/settings.json`, note its contents. Starting with Aspire 13.2, all AppHosts use `aspire.config.json` instead, so the existing `.aspire/settings.json` will no longer be used. Back it up for reference:

   <OsAwareTabs syncKey="terminal">
     <Code
       slot="unix"
       lang="bash"
       code="cp .aspire/settings.json .aspire/settings.backup.json"
     />
     <Code
       slot="windows"
       lang="powershell"
       code="copy .aspire\settings.json .aspire\settings.backup.json"
     />
   </OsAwareTabs>

</Steps>

## Migrate to TypeScript

With your inventory complete, follow these steps to create the TypeScript AppHost.

<Steps>

1. **Rename or move your existing `AppHost.cs`**

   Don't delete it—keep it as a reference while migrating:

   <OsAwareTabs syncKey="terminal">
     <Code
       slot="unix"
       lang="bash"
       code="mv AppHost.cs AppHost.cs.bak"
     />
     <Code
       slot="windows"
       lang="powershell"
       code="ren AppHost.cs AppHost.cs.bak"
     />
   </OsAwareTabs>

   If you have a project-based AppHost, also rename the `.csproj` file:

   <OsAwareTabs syncKey="terminal">
     <Code
       slot="unix"
       lang="bash"
       code="mv AppHost.csproj AppHost.csproj.bak"
     />
     <Code
       slot="windows"
       lang="powershell"
       code="ren AppHost.csproj AppHost.csproj.bak"
     />
   </OsAwareTabs>

2. **Initialize the TypeScript AppHost**

   From your AppHost directory, run:

   ```bash title="Initialize TypeScript AppHost"
   aspire init --language typescript
   ```

   Or if you prefer the interactive prompt:

   ```bash title="Initialize with interactive prompt"
   aspire init
   ```

   When prompted, select **TypeScript (Node.js)**:

   ```text
   Select AppHost Language
   Choose the programming language for your Aspire AppHost.
   This selection will be saved for future use.

   Which language would you like to use?

     C# (.NET)
   > TypeScript (Node.js)

   (Type to search)
   ```

   <LearnMore>
   See the [aspire init command reference](/reference/cli/commands/aspire-init/).
   </LearnMore>

   This creates the following structure:

   <FileTree>

   - workspace/
     - .modules/  Generated TypeScript SDK
     - apphost.ts  Your new AppHost
     - aspire.config.json  AppHost configuration (replaces .aspire/settings.json)
     - package.json
     - tsconfig.json
     - AppHost.cs.bak  Your reference file

   </FileTree>

   <Aside type="note">
   Starting with Aspire 13.2, all AppHosts use `aspire.config.json` in the project root instead of `.aspire/settings.json`. Integration packages and other configuration are managed in this file.
   </Aside>

   <LearnMore>
   Browse the [TypeScript API reference](/reference/api/typescript/) to explore available methods and types.
   </LearnMore>

3. **Add your integrations**

   For each package you noted earlier, run `aspire add` with the full package name:

   ```bash title="Add integrations"
   aspire add Aspire.Hosting.Redis
   aspire add Aspire.Hosting.PostgreSQL
   aspire add Aspire.Hosting.NodeJs
   ```

   Each `aspire add` command updates `.aspire/settings.json` and regenerates the TypeScript SDK in `.modules/`.

   <LearnMore>
   See the [aspire add command reference](/reference/cli/commands/aspire-add/).
   </LearnMore>

4. **Translate your resources to TypeScript**

   Map your C# resource definitions to their TypeScript equivalents. The key differences are:

    - TypeScript APIs are `async` and require `await`
    - Method names use camelCase instead of PascalCase
    - `builder.Build().Run()` becomes `builder.build()` then `app.run()` (both awaited separately)
    - Project references use `addProject('name', '../path')` instead of `AddProject<Projects.Name>("name")`

   **Compare before `apphost.cs` to after `apphost.ts`**:

   <Tabs syncKey="apphost-lang">
   <TabItem label="C#">

   ```csharp title="AppHost.cs"
   var builder = DistributedApplication.CreateBuilder(args);

   var cache = builder.AddRedis("cache");
   var db = builder.AddPostgres("pg")
       .AddDatabase("catalog");

   var api = builder.AddProject<Projects.Api>("api")
       .WithReference(cache)
       .WithReference(db);

   builder.AddProject<Projects.Web>("web")
       .WithReference(api)
       .WithExternalHttpEndpoints();

   builder.Build().Run();
   ```

   </TabItem>
   <TabItem label="TypeScript">

   ```typescript title="apphost.ts"
   import { createBuilder } from './.modules/aspire.js';

   const builder = await createBuilder();

   const cache = await builder.addRedis('cache');
   const db = await builder.addPostgres('pg')
     .addDatabase('catalog');

   const api = await builder.addProject('api', '../api')
     .withReference(cache)
     .withReference(db);

   await builder.addProject('web', '../web')
     .withReference(api)
     .withExternalHttpEndpoints();

   const app = builder.build();
   await app.run();
   ```

   </TabItem>
   </Tabs>

</Steps>

## Common mappings

The following table shows common C# AppHost patterns and their TypeScript equivalents:

| C# Pattern | TypeScript Equivalent |
| --- | --- |
| `builder.AddRedis("name")` | `await builder.addRedis('name')` |
| `builder.AddPostgres("name")` | `await builder.addPostgres('name')` |
| `.AddDatabase("db")` | `.addDatabase('db')` |
| `builder.AddProject<Projects.Api>("api")` | `await builder.addProject('api', '../api')` |
| `.WithReference(resource)` | `.withReference(resource)` |
| `.WithExternalHttpEndpoints()` | `.withExternalHttpEndpoints()` |
| `.WithHttpEndpoint(port: 3000)` | `.withHttpEndpoint({ port: 3000 })` |
| `.RunAsEmulator()` | `.runAsEmulator()` |
| `builder.AddAzureStorage("storage")` | `await builder.addAzureStorage('storage')` |
| `builder.AddNodeApp("name", "../path", "file.js")` | `await builder.addNodeApp('name', '../path', 'file.js')` |
| `builder.Build().Run()` | `builder.build()` then `await app.run()` |

<Aside type="note">
The TypeScript APIs are generated from the hosting integrations you've added. Use your IDE's autocomplete to discover available methods and parameters—the generated types in `.modules/` provide full IntelliSense.
</Aside>

## Verify the migration

After translating your resources, run the TypeScript AppHost:

```bash title="Run TypeScript AppHost"
aspire run
```

<LearnMore>
See the [aspire run command reference](/reference/cli/commands/aspire-run/).
</LearnMore>

Verify that:

- All resources start correctly in the Aspire Dashboard
- Service discovery works between resources
- Container resources (Redis, PostgreSQL, etc.) are running
- Your application projects connect to their dependencies

## Clean up

Once you've verified the migration is successful:

<Steps>

1. Remove the backup files:

   <OsAwareTabs syncKey="terminal">
     <Code
       slot="unix"
       lang="bash"
       code={"rm AppHost.cs.bak\nrm AppHost.csproj.bak"}
     />
     <Code
       slot="windows"
       lang="powershell"
       code={"del AppHost.cs.bak\ndel AppHost.csproj.bak"}
     />
   </OsAwareTabs>

2. Remove any leftover C# build artifacts:

   <OsAwareTabs syncKey="terminal">
     <Code
       slot="unix"
       lang="bash"
       code={"rm -rf bin\nrm -rf obj"}
     />
     <Code
       slot="windows"
       lang="powershell"
       code={"rmdir /s /q bin\nrmdir /s /q obj"}
     />
   </OsAwareTabs>

3. Commit the migrated AppHost to source control.

</Steps>
