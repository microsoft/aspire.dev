## Name

`aspire init` - Initialize Aspire support in an existing codebase with a C# or TypeScript AppHost.

## Synopsis

```bash title="Aspire CLI"
aspire init [options]
```

## Description

The `aspire init` command initializes Aspire support in an existing repo or workspace. It scaffolds a minimal **C# AppHost** or **TypeScript AppHost** skeleton together with an `aspire.config.json`, then hands off all further wiring to the **`aspireify` agent skill**.

The `aspireify` skill is a one-time setup skill that an AI coding agent (such as GitHub Copilot or Claude Code) runs immediately after the skeleton is dropped. The skill handles the heavy lifting that previously happened inside `aspire init` itself:

- Scanning the repo and discovering existing projects, services, and containers
- Wiring resources into the AppHost with `WithReference`, `WaitFor`, endpoints, and volumes
- Adding ServiceDefaults and configuring OpenTelemetry for each service
- Validating the wiring with a smoke-test `aspire start` before removing itself

This command defaults to **interactive** mode. When executed without any options, the command prompts you for the AppHost language when more than one path is available.

<Aside type="note">
The `--source` and `--version` options are deprecated and no longer affect `aspire init`. They are accepted for backward compatibility but will be removed in a future release.
</Aside>

## Options

The following options are available:

- **`--channel`**

  Channel to use for templates (`stable`, `staging`, `daily`).

- **`--language`**

  The AppHost language to scaffold (`csharp`, `typescript`).

## The aspireify skill

After `aspire init` drops the skeleton files, it installs the `aspireify` skill into your AI agent's skill directory (the same directory used by `aspire agent init`). The skill is a Markdown file with instructions that guide your AI coding agent through completing the Aspire wiring for your specific codebase.

The skill covers both **C# AppHosts** and **TypeScript AppHosts** and includes built-in guidance for:

| Scenario | Covered by skill |
|---|---|
| Multi-project .NET solutions (`.slnx`) | ✓ |
| Docker Compose migration | ✓ |
| JavaScript/TypeScript frontends (Vite, Node.js, monorepos) | ✓ |
| Python and Go services | ✓ |
| OpenTelemetry / ServiceDefaults setup | ✓ |
| Persistent containers and volumes | ✓ |
| Secrets and parameterized configuration | ✓ |

The skill self-deletes from the skill directory after a successful `aspire start`, leaving your repo clean.

## Examples

- Initialize Aspire support in the current repo interactively:

  ```bash title="Aspire CLI"
  aspire init
  ```

- Initialize Aspire support with a TypeScript AppHost:

  ```bash title="Aspire CLI"
  aspire init --language typescript
  ```

- Initialize Aspire support using the `daily` channel templates:

  ```bash title="Aspire CLI"
  aspire init --channel daily
  ```
