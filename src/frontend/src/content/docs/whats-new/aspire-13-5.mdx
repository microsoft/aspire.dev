Aspire 13.5 arrives with a focused set of improvements to the **Visual Studio Code extension** (v1.14.0), including **Bun debugging support**, richer **parameter display** in the resource tree and AppHost CodeLens, security hardening against **shell injection**, and updated npm dependencies to close open security advisories.

We'd love to hear what you think. Drop by [<Icon name="discord" class='d-inline' /> Discord](https://aka.ms/aspire-discord) to chat with the team and the community, or file feedback and issues on [<Icon name="github" class='d-inline' /> GitHub](https://github.com/microsoft/aspire/issues).

## 🧰 VS Code extension

The Aspire extension for Visual Studio Code is updated to **v1.14.0** in Aspire 13.5.

### Bun debugging support

The extension now integrates with the [Bun](/integrations/frameworks/bun-apps/) hosting integration introduced in Aspire 13.4. When you start an AppHost that includes a Bun service in debug mode, the extension attaches the Bun debugger automatically, giving you breakpoints, stepping, and variable inspection in Bun-based services just as you would with Node.js or C# projects.

### Improved parameter display

The resource tree and AppHost CodeLens now present parameters in a more informative and privacy-conscious way:

- **Secrets are masked** — parameter values marked as secrets no longer appear in plain text in the sidebar or CodeLens hover.
- **Long values are truncated** — parameters with lengthy values are truncated with an ellipsis so the resource tree remains scannable.
- **Missing values are shown explicitly** — when a required parameter has no value assigned yet, the resource tree calls it out rather than leaving an empty field that could be mistaken for a blank default.

### Security hardening and dependency updates

The extension routes all Aspire CLI arguments through structured shell quoting before sending them to the integrated terminal. This closes a shell injection vector that could be triggered by crafted resource names or parameter values.

In addition, the following npm dependencies were updated to resolve open security advisories:

- `undici` — addressed a vulnerability flagged by Dependabot.
- `ws`, `fast-uri`, `qs`, `@nevware21/ts-utils` — updated to their latest patched versions.

### Other fixes

- Fixed excessive AppHost discovery requests that could flood the workspace with redundant file-system scans, reducing unnecessary I/O during idle sessions.
- The Aspire pane now shows a compatibility error when the running AppHost returns empty `describe` output, rather than silently displaying an empty resource list.
