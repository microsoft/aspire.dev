The `loc-breakdown` canvas is a project-scoped [Copilot CLI canvas extension](https://docs.github.com/en/copilot/using-github-copilot/coding-agent/using-copilot-coding-agent-in-your-ide) included in the Aspire repository under `.github/extensions/loc-breakdown/`. It shows a breakdown of the lines of code changed on the current branch, grouped by project or category, with a heuristic characterization of each group.

Because it lives under `.github/extensions/`, the canvas is automatically available to anyone working inside the Aspire repository with Copilot CLI — no installation step is needed.

## What the canvas shows

When opened, the canvas:

1. Resolves the merge-base of the current branch against `origin/HEAD` (falling back to `main` or `master`).
2. Runs `git diff --numstat` between that merge-base and `HEAD`.
3. Groups changed files into buckets that reflect the Aspire repository layout — each `src/<Project>`, `src/Components/<Component>`, `tests/<Suite>`, `playground/<App>`, and `tools/<Tool>` gets its own row, plus catch-all buckets for `docs`, `eng`, `.github`, the VS Code extension, and `.agents`.
4. Sorts categories by total lines touched (additions + deletions), descending.
5. Tags each category with a short characterization:

| Tone | When it applies |
|------|-----------------|
| Mechanical | Localization files, generated files, snapshots, assets, and trivial edits |
| Documentation | Files that are exclusively `.md`, `.mdx`, or `.txt` |
| Detailed | Tests, small additions, and moderate edits |
| Careful | CI workflows, significant new code, large deletions, and big refactors |

Each row is expandable to show the per-file added/removed counts within a category.

## Opening the canvas

In any Copilot CLI session inside the Aspire repository, ask the agent to open the **LOC breakdown** canvas. For example:

> "Open the LOC breakdown canvas."
> "Show me a per-project summary of my diff."

The canvas also exposes a `refresh` action that the agent can invoke to recompute the report after new commits are added.

### Optional inputs

| Input | Default | Description |
|-------|---------|-------------|
| `cwd` | Agent's current working directory | Working directory inside a git repo |
| `base` | `origin/HEAD` | Base ref to diff against |
| `head` | `HEAD` | Head ref to diff |

<Aside type="note">
User-supplied `base` and `head` values are validated before being passed to git to prevent option injection. Values must not start with `-` and are restricted to safe characters.
</Aside>

## Refreshing the report

Use the **↻ Refresh** button in the canvas UI, or ask the agent to refresh the canvas, to recompute the report after making additional commits.
